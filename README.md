# ケンポアシスト（KenpoAssist）

**健保事務のAI相棒**

健康保険組合の事務担当者が、被保険者からの問い合わせに対する回答草案・参考文献を
AIで作成し、AIと相談しながら最終回答に仕上げ、対応履歴を管理するためのローカルアプリです。

利用するAIは **Claude / ChatGPT / Gemini** から選べます。各AIは、**利用者ご自身が契約・ログインした
公式CLI** を通じて呼び出します（APIキーの設定は不要）。

---

## 特長 / 設計方針

- **ローカル完結**：問い合わせ・回答・履歴はすべてお使いのPC内（`kenpo_support.db`）に保存され、
  外部サーバーには送信されません。
- **AIは3社から選択**：画面右上のセレクタでいつでも切り替え可能。導入済みのCLIのみ選択できます。
- **インストール型・買い切り**：1台のPCにインストールして利用します。

> ⚠️ **クラウドAIへの送信について**
> 回答草案の生成や相談の際、問い合わせ本文はお選びのAI事業者（Anthropic / OpenAI / Google）の
> クラウドへ送信されます。被保険者の個人情報を含めて入力する場合は、各社の利用規約・プライバシー
> ポリシーをご確認のうえ、貴組合の規程に従ってご利用ください。

---

## 動作要件

- 利用したいAIの公式CLI（下記「導入手順」）が **インストール＆ログイン済み** であること
- インターネット接続
- ソースから動かす場合のみ Python 3.10 以降（配布版exeはPython不要）

---

## 導入手順（利用者向け・配布版exe）

配布版（単一実行ファイル）を受け取った場合は Python のインストールは不要です。

1. 下記「利用するAIのCLIを準備」を済ませる
2. `KenpoAssist`（Windowsは `KenpoAssist.exe`）をダブルクリックで起動
3. 自動でブラウザに画面が開きます（`http://127.0.0.1:8765/`）

> データ（問い合わせ履歴・設定）は `ユーザーのホーム/.kenpoassist/kenpo_support.db` に保存されます。

### 利用するAIのCLIを準備（使う分だけでOK）

| 選択肢 | 必要なCLI | ログイン |
|--------|-----------|----------|
| Claude   | Claude Code CLI（`claude`） | `claude` を起動し、ご自身のClaudeアカウントでログイン |
| ChatGPT  | OpenAI Codex CLI（`codex`） | `codex` を起動し、ご自身のChatGPTアカウントでログイン |
| Gemini   | Gemini CLI（`gemini`）       | `gemini` を起動し、ご自身のGoogleアカウントでログイン |

> 画面右上のAIセレクタでは、**インストール済みのCLIだけ**が選択可能になります
> （未導入のものは「（未導入）」と表示されます）。

## ソースから起動（開発者向け）

```bash
pip install -r requirements.txt
python run.py
```

既定ブラウザで `http://127.0.0.1:8765/` が自動的に開きます。

---

## 配布パッケージの作成（ビルド／開発者向け）

PyInstaller で Python 同梱の単一実行ファイルを生成します。
**Windows向けexeはWindows上で、macOS向けはmac上で**ビルドしてください（クロスビルド不可）。

```bash
# macOS / Linux
./build_unix.sh

# Windows
build_windows.bat
```

生成物は `dist/KenpoAssist`（Windowsは `dist\KenpoAssist.exe`）です。
これを購入者に配布します。`static/` 等は実行ファイルに同梱されます
（AIのCLIは同梱されず、利用者が各自で用意します）。

## ライセンス管理（販売者向け）

本製品はオフライン署名方式のライセンスキーで保護されています。
購入者は初回起動時にキーを入力して有効化します（サーバー不要・オフラインで検証）。

### 初回のみ：鍵ペアの生成
```bash
python issue_license.py --genkeys
```
- 出力された**公開鍵**を `licensing.py` の `PUBLIC_KEY_B64` に貼り付けてからビルドします。
- **秘密鍵 `license_private_key.txt` は販売者だけが厳重に保管**してください
  （`.gitignore` 済み・配布物には絶対に含めない）。

### 販売のたびに：ライセンスキーの発行
```bash
python issue_license.py --name "○○健康保険組合" --expires 2027-03-31 --seats 1
# 無期限の場合は --expires を省略
```
出力されたキーを購入者にお渡しします。

> ⚠️ **限界の理解**：ローカルで動く製品のため、キー検証も購入者の手元にあります。
> 善意の無断コピー・共有の抑止が目的で、完全なコピー防止ではありません。

## 使い方

1. **新規問い合わせ**：カテゴリ・担当者・問い合わせ内容を入力し「AI回答草案を生成」。
   回答文と参考文献・根拠が生成されます。
2. **AIと相談**：草案について不明点をAIに質問できます。
3. **最終回答へ反映**：相談結果を踏まえ、元の草案を土台に最終回答を再生成します。
4. **保存／履歴**：対応中・完了として保存し、「対応履歴」から検索・編集できます。

## 重要：ライセンスとご利用範囲（必ずお読みください）

本製品は **個人の通常利用** を前提に、各AIの個人向けプラン（Claude Pro/Max・ChatGPT Plus 等・
Gemini）を、利用者ご自身のログインで呼び出します。

- 各AI事業者の個人向けプランの利用規約は **個人の通常利用** を想定しています。
  **法人での共有利用・複数名による常時稼働・自動化された大量呼び出し** は規約に抵触する
  おそれがあります。複数の担当者で利用する場合は、**担当者ごとに本製品をインストールし、
  各自のアカウントで** ご利用ください（席単位）。
- 業務用途で常時・大量に利用される場合は、各社の **API（商用条件）** への切替をご検討ください。
- 各AIには利用上限（レート制限）があります。上限到達時はエラーが表示されます。

## 既知の留意点

- **CLIの仕様変更**：各社CLIの呼び出し方法・出力形式は変更される場合があります。
  呼び出しコマンドは `main.py` の `PROVIDERS` に集約しているため、必要に応じて調整できます。
- **周囲設定の隔離（対応済み）**：`claude` 等のCLIは実行環境のグローバル設定
  （`~/.claude/CLAUDE.md` 等）やスキルを読み込み、出力に混入することがあります。
  本製品では Claude を `--safe-mode`（全カスタマイズ無効・サブスク認証は維持）で呼び出し、
  かつ全プロバイダを中立な一時ディレクトリで実行することで、これらの混入を防いでいます。

## 設定

`.env.example` を `.env` にコピーして編集できます（任意）。

| 変数 | 既定 | 説明 |
|------|------|------|
| `DEFAULT_AI_PROVIDER` | `claude` | 初期選択のAIプロバイダ |
| `AI_TIMEOUT` | `120` | AI応答のタイムアウト秒数 |
| `PORT` | `8765` | 待受ポート |
